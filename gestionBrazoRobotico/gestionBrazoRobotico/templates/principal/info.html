informacion General

Antecedentes

Este proyecto surgió como una iniciativa para abordar la recolección de residuos, centrándose en materiales reciclables. Nuestra meta principal fue desarrollar un brazo robótico capaz de ofrecer una solución integral a este desafío. Buscábamos no solo mejorar la eficiencia de la recolección, sino también promover una gestión sostenible de desechos. El proyecto fue un éxito, logrando crear un sistema que integra la recolección manual. Mediante el uso de pulsadores, se facilitó el control manual del brazo robótico, permitiendo una operación intuitiva y eficaz.

Problemática

En este proyecto, continuaremos mejorando el brazo robótico. En este momento, nos enfrentamos al desafío de hacer que el brazo pueda funcionar de manera remota o a distancia. Esto implica la capacidad de operar el brazo sin necesidad de estar físicamente presentes en el lugar donde se encuentra. Esta funcionalidad es crucial en situaciones de emergencia que nos impidan acceder al entorno físico. Por lo tanto, nuestro próximo paso es buscar soluciones para el control remoto del brazo.

Propuesta

Conscientes al desafío a afrontar, se llego a la siguiente solución: manejar el dispositivo mediante una aplicación o una página web, donde además de poder controlarlo de forma remota, tendrá incorporada una cámara para captar el entorno de manera virtual.

Objetivos

  • Identificar las funcionalidades que va a proporcionar la página al usuario para que este tenga una experiencia satisfactoria cumpliendo con sus expectativas, debido a ello se recurrirá a lo que se llaman historias de uso y casos de uso.
  • Crear un prototipo de la interfaz que se va a implementar para el manejo del brazo robótico, con la meta de hacerla intuitiva para el usuario para que este no cometa errores que puedan llegar a darle una mala experiencias.

Requerimientos

Para programación orientado a objetos se requiere:

  • Uso de los conocimientos básicos para el desarrollo de la lógica de la aplicación Web utilizando varias tecnologías, en especial Django
  • Conocimientos adquiridos de los diagramas de clases y los casos de uso
  • Implementación correcta de la interfaz, incluyendo las clases (Modelos en Django) descritas en el diagrama de Clases
  • Manejo de buenas prácticas de programación para un desarrollo más eficiente y mantenible

Para Diseño de Circuitos:

  • Generar un buen uso del hardware del circuito por medio de los conocimientos adquiridos.

Materiales

Placa ESP32-cam

La Esp 32 Cam es una placa de la familia ESP32 de bajo costo, que incluye una cámara integrada de tamaño pequeño, ideal para la aplicación y construcción de proyectos de bricolaje. Esta placa además integra WIFI, bluetooth y BLE de bajo consumo, cuenta con 2 CPU LX6 de 32 bits de alto rendimiento. Conforma una arquitectura de tubería de 7 etapas, incluso cuenta con sensores de algunos tipos, en tanto que su frecuencia principal varía de 80 MHz a 240 MHz.

Servomotores

Son los componentes que darán movimiento al brazo robótico, se trata de motores que brindan un mayor control que los motores de corriente directa convencionales. Utilizan el ancho de pulso para funcionar y su consumo varía dependiendo del modelo. Para este proyecto hemos utilizado dos modelos de servomotores

Micro servo sg90

Micro servo mg90

USB to TTL Serial Cable

Es una herramienta muy útil que permite conectar la computadora con cualquier dispositivo serial TTL a traves de del puerto USB, Nosotros lo utilizamos para cargar el programa que define el movimiento de brazo haciendo uso de los datos que llegan desde otro programa que en este caso es el apartado de controles en nuestra página web

Protoboard

Es una placa de pruebas que nos permite interconectar de manera temporal todos los componentes del brazo robótico permitiendo la realización de pruebas de funcionamiento así como presentaciones del proyecto. consiste en una placa de plástico con orificios interconectados de manera conveniente y ordenada para realizar proyectos electrónicos; los orificios siguen patrones específicos como por ejemplo las líneas de alimentación

Cables Jumper

Son conectores con recubrimiento plástico que permiten cerrar un circuito para hacerlo funcionar. Se utilizan para verificar y modificar el funcionamiento de alguna pieza de hardware. Los utilizaremos para relizar las respectivas conexiones entre los componentes del brazo robótico

Plataformas y lenguajes utilizados

Django: Python Framework

Django es un framework de desarrollo web de código abierto, escrito en Python. Sigue el principio de DRY (Don't Repeat Yourself) y se basa en el patrón de diseño MVC (Modelo Vista Controlador). Django es un framework de alto nivel que fomenta el desarrollo rápido y limpio, con un diseño pragmático y una sintaxis sencilla.

Lo hemos utilizado para el desarrollo backend de la página web, que muestra la información y da acceso a los controles del brazo Robótico

HTML

HTML (HyperText Markup Language) es el lenguaje de marcado estándar para la creación de páginas web. Define la estructura de una página web y su contenido, como texto, imágenes, videos, formularios, etc.

CSS

CSS (Cascading Style Sheets) es un lenguaje de diseño que se utiliza para dar estilo a las páginas web. Permite controlar la apariencia de los elementos de una página, como el color, la fuente, el tamaño, el espaciado, etc.

JavaScript

JavaScript es un lenguaje de programación que se utiliza para crear interactividad en las páginas web. Permite añadir funcionalidades dinámicas, como animaciones, efectos visuales, validación de formularios, etc.

Este poderoso lenguaje fue utilizado para implementar los controles del brazo robótico y dar algunas funcionalidades menores a nuestra página como algunas animaciones

Arduino IDE

Arduino IDE es un entorno de desarrollo integrado (IDE) que se utiliza para programar placas Arduino. Proporciona un editor de código, un compilador y un cargador de arranque para cargar el código en la placa Arduino.

Lo hemos utilizado para programar el microcontrolador de la placa ESP32-CAM, haciendo que adopte el comportamiento deseado
